USEFUL H2 QUERIES
-----------------
SELECT * FROM TENANTS;
SELECT * FROM APP_USERS;
SELECT * FROM SYSTEM_ADMINS;
SELECT * FROM REFRESH_TOKENS;
SELECT * FROM PASSWORD_RESET_TOKENS;
SELECT * FROM AUDIT_LOGS;

View latest audit logs:
SELECT * FROM AUDIT_LOGS ORDER BY CREATED_AT DESC;

View tenant users:
SELECT * FROM APP_USERS WHERE TENANT_ID = '<tenant-id>';

Reactivate a tenant during local testing:
UPDATE TENANTS SET STATUS = 'ACTIVE' WHERE ID = '<tenant-id>';

Reactivate a user during local testing:
UPDATE APP_USERS SET STATUS = 'ACTIVE' WHERE ID = '<user-id>';


WHEN TO RESET LOCAL H2 DATABASE
-------------------------------
For local development, reset H2 when enum values or table structure cause old schema issues.

Steps:
1. Stop the app.
2. Delete the data/ folder.
3. Restart the app.
4. Re-onboard a tenant and login again.

Common reason:
H2 check constraints may still contain old enum values.
