# Authentication difficulty

```text

↓

↓

↓

↓

```

This is not too hard once you understand the flow.

The hard part is not the logic. The hard part is understanding where Spring Boot expects each part to be placed.

# Permission system difficulty

A simple role system is easy:

```text
SUPER_ADMIN
TENANT_ADMIN
MANAGER
STAFF
VIEWER
```

You can protect APIs like:

```java
@PreAuthorize("hasRole('TENANT_ADMIN')")
@GetMapping("/users")
public List<UserDto> getUsers() {
    return userService.getUsers();
}
```

But a real permission system is harder.

For example:

```text
CREATE_USER
UPDATE_USER
DELETE_USER
VIEW_BILLING
MANAGE_SUBSCRIPTION
VIEW_AUDIT_LOGS
EXPORT_DATA
```

Then roles contain many permissions.

Example:

```text
TENANT_ADMIN → CREATE_USER, UPDATE_USER, VIEW_BILLING, MANAGE_SETTINGS
MANAGER → CREATE_PROJECT, UPDATE_PROJECT, VIEW_REPORTS
STAFF → VIEW_TASKS, UPDATE_TASKS
VIEWER → VIEW_ONLY
```

This requires more tables:

```text
users
roles
permissions
user_roles
role_permissions
```

This is manageable, but you need good database modeling.

# Tenant isolation difficulty

This is the most important and most dangerous part.

Every tenant-owned table must have a `tenant_id`.

Example:

```text
users
projects
subscriptions
invoices
files
audit_logs
notifications
```

Each API must only return records belonging to the current tenant.

Bad query:

```java
projectRepository.findAll();
```

Safe query:

```java
projectRepository.findByTenantId(currentTenantId);
```

The risk is that one forgotten filter can expose another tenant’s data.

That is why tenant isolation is harder than authentication.

---

# Recommended beginner-friendly tenant isolation strategy

Do **not** start with separate database per tenant.

Use:

```text
Single PostgreSQL database
Shared schema
tenant_id column in every tenant-owned table
```

Example:

```java
@ManyToOne
@JoinColumn(name = "tenant_id", nullable = false)
private Tenant tenant;
```

Then enforce filtering through service methods:

```java
public List<Project> getProjectsForCurrentTenant() {
    UUID tenantId = currentUserService.getCurrentTenantId();
    return projectRepository.findByTenantId(tenantId);
}
```

This is the easiest and safest architecture for your learning stage.
